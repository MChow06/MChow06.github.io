Prompt 03 · Dynamic Prediction Engine
Generate Product Names From Any Keyword
Add JavaScript logic: If a keyword trend is high, generate a predicted product name. Example: "vanilla" → "Vanilla Body Butter". Display a confidence percentage. Make it dynamic.
What this built
Replaced every hardcoded product prediction with a fully algorithmic engine. No lookup tables — just pure composable logic that works on any keyword string. Also added the interactive Prediction Lab where users can type anything and get a real generated prediction.
Prediction Lab Section parseKeyword() function calcConfidence() formula getModifier() picker Typewriter Animation Animated Confidence Ring Quick-fill Chips → Predict buttons on cards
parseKeyword(): tokenizes any string, scans against 40 scent anchors (vanilla, cedar, rose…) and 25 product-type signals (butter, mist, candle…) to extract the product form
calcConfidence(): weighted formula — mentions × 0.35 + velocity × 0.25 + engagement × 0.20 + searchLift × 0.12 + creatorPosts × 0.08 — normalized to 52–96%
getModifier(): picks a deterministic adjective (Deluxe, Luxe, Signature, Essential…) based on confidence tier and a hash of the scent string — same keyword always gives the same modifier
Product name is assembled as: Scent Anchor + Modifier + Product Type — e.g. "Cedar Signature 3-Wick Candle"
⚗️
Result: Fully dynamic — works on any keywordType "bergamot scrub", "pumpkin candle", or "ocean cologne" — the engine handles it all without any prewritten answers.

Prompt 05 · Deployment
Deploy It as a Public Website
I want to deploy this as a public website using GitHub Pages. Give me step-by-step instructions from creating an account to making the site live.
What this produced
A complete deployment walkthrough — no servers, no hosting fees, no DevOps. GitHub Pages serves the static HTML file with global CDN and HTTPS included for free. The site went from a local file to a publicly accessible URL in under 5 minutes.
GitHub Account Setup Repository Creation File Upload via Browser GitHub Pages Config Live HTTPS URL Custom Domain Option
Repository must be named exactly username.github.io — this is the special name that activates GitHub Pages as a root site
The uploaded file must be named exactly index.html — GitHub Pages looks for this filename at the root URL
Pages must be enabled in Settings → Pages by setting the branch to main — takes about 60 seconds to build
Cost: $0. GitHub Pages is completely free for public repositories with no bandwidth limits for normal traffic
🌐
Result: Live at mchow06.github.ioThe site is publicly accessible worldwide, served over HTTPS with GitHub's CDN. No monthly fees, no server to maintain.
